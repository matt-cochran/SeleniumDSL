Selenium DSL

Domain Specific Language for
Executing Selenium Tests

View project onGitHub

Status:

Alpha. Parser/Executor is working for the syntax (below). More operations and better output is required before we can go to beta.

Goal:

This project's goal is to create tests in a meta-language where tests can be written and understood in a syntax closer to english.

Overview:

This project is an attempt to make maintenance of automated browser tests easier for web applications.

When the code in a project matures, the amount of tests grows and changes become more complex and potentially have a broader impact. As a developer I want to be able to offload some of the test maintenance and creation to the qa and/or business so I can spend more time on building cool new features. Possible opportunities would be the high-level tests like of defining critical paths through the site or new features. These tests that will probably live the longest because they have the least coupling with implementation. Some of the more technical tests that are a bit closer to the metal will always be maintained by developers, but you don't need to be a developer to understand the high-level business requirements and script basic tests against the application.

Sample Syntax:

Here is the syntax for a test scenario:

test the home page.
show that the page loads.
use firefox, ie, chrome.
define target page as http://www.google.com.
define search box as element named 'q'.
{
    go to the target page.
    clear the search box.
    send 'asdf' to the search box.
    assert the search box has value 'asdf'
}

The first line will define the target page for the story:

test the home page.

The second line defines what we are testing:

show that the page loads.

The third line will define the browsers we want to test the story against.

use firefox, ie, chrome.

The next lines will define variables I can push into the test steps

define target page as http://www.google.com.

define search box as element named 'q'.

And finally the test steps are embedded in brackets:

{
    go to the target page.
    clear the search box.
    send 'asdf' to the search box.
    assert the search box has value 'asdf'
}

Project: MC.Selenium.DSL

this project extends the OpenQA.Selenium.IWebDriver by adding a .ExecuteCommand() method that you can pass a test step, or series of test steps to.

You can combine commands by putting them in order and optionally separating them with a period, comma, or 'then' text. The following would be valid test instructions:

assert select with id 'y' has value 'one',
assert option with id 'opt1' is selected
then assert option with id 'opt2' is not selected.

set option with id 'opt2' to selected,
then assert select with id 'y' has value 'two' assert option with id 'opt1' is not selected
assert option with id 'opt2' is selected.

set option with id 'opt2' to not selected,
assert select with id 'y' has value ''
